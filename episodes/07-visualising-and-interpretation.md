:::::::::::::::::::::::::::::::::::::: questions

- How can we visually inspect the evidence behind a genomic result?
- How do we interpret the found results?

::::::::::::::::::::::::::::::::::::::::::::::::

::::::::::::::::::::::::::::::::::::: objectives

- Open an alignment and variant file in IGV or an equivalent viewer. 
- Locate a region of interest such as cyp51A. 
- Open a Newick tree in a tree viewer. 
- Compare visual evidence with tabular workflow outputs. 

::::::::::::::::::::::::::::::::::::::::::::::::

## Introduction

##  Practical

Practical stations
1.	Genome browser view
Open BAM, BAI, VCF, and reference files. Navigate to the region of interest. 
2.	Tree view
Open the Newick tree. Colour or label tips by metadata such as source, date, or resistance status. 
3.	Table view
Open the MLSA table, variant annotation table, and resistance summary. 
4.	Cross-output interpretation
For one isolate, trace the story across all outputs: MLSA placement → SNP-tree position → resistance marker → visual evidence. 

::::::::::::::::::::::::::::::::::::: keypoints

- Visualisation is a quality and interpretation aid, not a replacement for the workflow output. 
- The same isolate should be traceable across tables, trees, and genome-browser views. 
- Discrepancies between outputs should trigger review rather than automatic acceptance.
